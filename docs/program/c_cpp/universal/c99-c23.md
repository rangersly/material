## 内联函数

避免函数调用开销(是否内联由编译器决定)

```c
inline int max(int a, int b) { return a > b ? a : b; }
```

## 指定初始化器

按照成员名初始化,更直观

```c
// 结构体
struct Point { int x, y; };
struct Point p = { .y = 10, .x = 5 };   // p.x = 5, p.y = 10

// 数组
int arr[5] = { [2] = 100, [4] = 200 };   // arr = {0,0,100,0,200}
```

## 复合字面量

临时创建匿名对象,用于函数参数或赋值

```c
draw_point((struct Point){ .x = 3, .y = 7 });
int *p = (int[]){ 1, 2, 3 };   // 指向数组第一个元素
```

## 可变参数宏

让宏接受不定参数

```c
#define debug(fmt, ...) printf("DEBUG: " fmt, __VA_ARGS__)
debug("x = %d, y = %d\n", x, y);
```

## restrict

告诉编译器指针独享指向的内存区域,可直接从寄存器读取

**对频繁存取的大数据提升效果明显(特别是本来就慢的堆区)**

> [!WARNING]
> 注意不要用在有多个指针指向的内存区域!!!

```c
void copy(int *restrict dest, const int *restrict src, size_t n) {
    for (size_t i = 0; i < n; i++) dest[i] = src[i];
}
```

## 柔性数组

结构体的最后一个成员可以是不完整数组,用于变长缓冲区

```c
struct Buffer {
    size_t len;
    char data[];    // 柔性数组,不占结构体大小
};

struct Buffer *buf = malloc(sizeof(struct Buffer) + 100);
buf->len = 100;
```

## 泛型编程

C11的泛型选择 `_Generic` 是一个编译时机制,它允许你根据表达式的类型,从多个候选项中 **选择** 一个

c语言本身是静态类型且不支持函数重载, `_Generic` 了一种 **类型泛型** 的技巧,与宏结合模拟函数重载

- 基本语法结构
  ```c
  _Generic(控制表达式,
         类型1: 表达式1,
         类型2: 表达式2,
         ...
         default: 默认表达式
  );
  ```
  - 控制表达式 : 只关心类型
  - 类型 : 完整的c类型名
  - 表达式 : 当控制表达式与类型标签匹配时替换整个 `_Generic` 为这个表达式
  - default分支 : 无匹配项时选择

- 示例
  ```c
  int abs_int(int x);
  double abs_double(double x);

  #define ABS(x) _Generic((x), \
                int: abs_int, \
                double: abs_double, \
                default: abs_int \
            )(x)
  ```

## typeof

编译期运算符

1. `typeof(类型名)` : 返回给定类型
2. `typeof(表达式)` : 返回表达式类型
3. `typeof_unqual(类型名/表达式)` : 类似,但移除顶层 `const` 等限定符

- **应用场景**
  1. 声明无初始化器的变量(auto必须有初始化器)
    ```c
    double calculateValue(void);
    // auto x = calculateValue(); // 合法
    typeof(calculateValue()) y;  // y 的类型是 double,无需初始化
    ```
  2. 简化复杂声明
    ```c
    // 原版
    double (*a[4])(int);
    //typeof重写
    typeof(double(int)) *a[4];
    ```
  3. 泛型宏
    ```c
    #define SWAP(a, b) do { \
    typeof(a) temp = (a); \
    (a) = (b); \
    (b) = temp; \
    } while(0)
    ```

## 标准属性

1. `[[deprecated]]` / `[[deprecated("原因")` 标记为弃用
  - 表示某个函数或变量依然允许使用,但不推荐继续使用
  ```c
  [[deprecated("Use new_function instead")]]
  void old_function() { /* ... */ }
  ```
2. `[[fallthrough]]` 有意穿透
  - 用于 `switch` 语句 `case` 标签,表明穿透 `case` 是本意
  ```c
  switch (n) {
    case 1:
        // 做些事
        [[fallthrough]]; // 明确表明意图，无警告
    case 2:
        // 做些别的事
        break;
  }
  ```
3. `[[nodiscard]]]` 返回值不可丢弃
  ```c
  [[nodiscard("Check the error code")]] int open_file(const char* name);
  
  void use() {
      open_file("data.txt"); // 编译器警告: 丢弃了 'nodiscard' 返回值
  }
  ```
4. `[[maybe_unused]]` 标记可能未使用,加在未使用实体前抑制警告
5. `[[noreturn]]` 标记不返回的函数
